# Quickstart

Get Verity running in three simple steps. You'll build a bootable ISO and test it in QEMU with port forwarding.

<Steps>
  <Step title="Clone the repository">
    Clone the Verity repository to your local machine:

    ```bash
    git clone https://github.com/aayushxr/verity.git
    cd verity
    ```
  </Step>

  <Step title="Build the ISO">
    Build the Verity ISO using the Makefile:

    <Tabs>
      <Tab title="Linux">
        On Linux, the build runs natively with root access:

        ```bash
        make build
        ```

        <Note>
        The build script requires root access to create chroots and mount filesystems. It will use `sudo` automatically.
        </Note>

        **Requirements:**
        - `bash`, `wget`, `xorriso`, `squashfs-tools`, `cpio`, `syslinux`
        - Root access (via sudo)
      </Tab>

      <Tab title="macOS">
        On macOS, the build runs inside Docker:

        ```bash
        make build
        ```

        The Makefile detects macOS and automatically runs the build inside an `alpine:3.21` container.

        <Note>
        Docker is required on macOS. The build runs in privileged mode to create the ISO.
        </Note>

        **Requirements:**
        - Docker
      </Tab>
    </Tabs>

    The build process takes 1-3 minutes and creates `build/verity.iso` (typically 50-100 MB).

    <Accordion title="What happens during the build?">
      The build script (`scripts/build.sh`) performs these steps:

      1. Downloads Alpine Linux minirootfs (3.21)
      2. Extracts and chroots into the rootfs
      3. Installs nginx, linux-lts, busybox-static
      4. Creates initramfs with kernel modules
      5. Copies configuration files (nginx.conf, sysctl.conf, init)
      6. Hardens the system (removes apk, shells, unnecessary tools)
      7. Creates read-only squashfs root
      8. Generates bootable ISO with ISOLINUX

      Total: ~400 lines of shell script.
    </Accordion>
  </Step>

  <Step title="Test with QEMU">
    Boot the ISO in QEMU with port forwarding:

    ```bash
    make test
    ```

    This runs the ISO in QEMU and forwards `http://localhost:8080` to port 80 inside the VM.

    <Note>
    QEMU is required for testing. Install `qemu-system-x86_64` if you don't have it.
    </Note>

    **Boot sequence:**
    1. BIOS loads ISOLINUX bootloader
    2. Kernel and initramfs boot
    3. Initramfs mounts squashfs root
    4. System init brings up networking
    5. nginx starts as PID 1

    After ~5 seconds, nginx should be running.
  </Step>

  <Step title="Verify it works">
    Open another terminal and test the server:

    ```bash
    curl http://localhost:8080
    ```

    You should see the default Verity landing page:

    ```html
    <!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="utf-8">
        <title>Verity</title>
        ...
    </head>
    <body>
        <div class="container">
            <h1>Verity</h1>
            <p>Serving static files. Replace www/ with your content.</p>
        </div>
    </body>
    </html>
    ```

    <Tip>
    The server is running from a read-only squashfs filesystem. There's no shell, no package manager, and no way to modify the system at runtime.
    </Tip>
  </Step>
</Steps>

## What you just built

You now have a bootable ISO that:

- **Boots in seconds** via ISOLINUX and a custom initramfs
- **Runs nginx as PID 1** — the only process in the system
- **Has a read-only root** via squashfs (no `remount,rw` possible)
- **Cannot be modified** at runtime — no package manager, no shell, no SSH
- **Is fully hardened** with kernel security via sysctl

The entire system is ~400 lines of shell that you can audit in one sitting.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build fails with 'Missing dependency'">
    Make sure you have all required tools installed:

    **Linux:**
    ```bash
    # Debian/Ubuntu
    sudo apt-get install bash wget xorriso squashfs-tools cpio syslinux

    # Arch Linux
    sudo pacman -S bash wget libisoburn squashfs-tools cpio syslinux
    ```

    **macOS:**
    ```bash
    # Install Docker Desktop
    brew install --cask docker
    ```
  </Accordion>

  <Accordion title="QEMU test doesn't start">
    Install QEMU:

    **Linux:**
    ```bash
    # Debian/Ubuntu
    sudo apt-get install qemu-system-x86

    # Arch Linux
    sudo pacman -S qemu
    ```

    **macOS:**
    ```bash
    brew install qemu
    ```
  </Accordion>

  <Accordion title="Can't connect to localhost:8080">
    1. Check that QEMU is running (you should see boot logs)
    2. Wait 5-10 seconds for nginx to start
    3. Look for "verity: starting nginx" in the boot output
    4. If you see errors, check the QEMU console output

    Common issues:
    - Port 8080 already in use: Change the port in `scripts/test.sh`
    - Firewall blocking: Check your firewall settings
  </Accordion>

  <Accordion title="Want to rebuild after changes">
    Clean the build directory and rebuild:

    ```bash
    make clean
    make build
    ```

    This removes `build/` and creates a fresh ISO.
  </Accordion>
</AccordionGroup>

## Command reference

```bash
make build    # Build the ISO (uses Docker on macOS)
make test     # Boot in QEMU with port 8080 forwarded
make clean    # Remove build artifacts
```

</Info>
