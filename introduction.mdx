# Verity

A sovereign, hardened Alpine Linux ISO that boots directly into nginx. No shell, no package manager, no SSH — just a static file server on a read-only root filesystem.

## What is sovereignty?

Most servers are held together by trust in things you don't control — upstream repos, package managers, update daemons, shell access that "probably nobody will use." Every one of those is an assumption, and every assumption is a surface.

**Sovereignty means your server runs exactly what you built, nothing more.**

Verity achieves this by eliminating every component that isn't strictly necessary to serve files:

<CardGroup cols={2}>
  <Card title="No package manager" icon="ban">
    apk is deleted after build. Nothing can be installed at runtime.
  </Card>
  <Card title="No shell" icon="terminal">
    No way to get an interactive session. No SSH, no TTY, no login.
  </Card>
  <Card title="No mutation" icon="lock">
    Root filesystem is squashfs — read-only by nature, not by policy. No `remount,rw`.
  </Card>
  <Card title="No ambient authority" icon="shield">
    Kernel module loading disabled after boot. ptrace restricted. kexec off.
  </Card>
</CardGroup>

## The result

A machine that does one thing, can't be told to do anything else, and can be fully understood by one person.

The entire system is **~400 lines of shell** across a handful of files. You can read all of it in one sitting.

## What Verity does

Verity builds a bootable ISO that:

- Boots via ISOLINUX into a custom initramfs
- Mounts a squashfs root filesystem (read-only)
- Brings up networking via DHCP
- Applies kernel hardening (sysctl)
- Runs nginx as PID 1 — nothing else

<Note>
The ISO is typically 50-100 MB and boots in seconds. nginx becomes PID 1, meaning if nginx stops, the system halts.
</Note>

## Use cases

Verity is designed for scenarios where you need:

- **Static file hosting** with maximum security
- **Immutable infrastructure** that can't be modified at runtime
- **Minimal attack surface** for internet-facing servers
- **Fully auditable systems** where you can verify every line of code
- **Sovereign servers** that run only what you built

<Warning>
Verity is not designed for general-purpose computing. It has no package manager, no shell, and no way to install software at runtime. If you need to change anything, rebuild the ISO.
</Warning>
